# API Keys Setup Guide

This guide walks you through getting all required API keys for the Life Journal MVP.

## 1. Claude API Key (Required)

### Get Your Key
1. Go to https://console.anthropic.com
2. Sign in or create account
3. Click "API keys" in sidebar
4. Click "Create new key"
5. Copy the key (starts with `sk-ant-`)

### Add to .env
```bash
CLAUDE_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxx
```

### Cost
- Free tier: ~$5 credit to start
- Pay-as-you-go: ~$0.80 per 1M input tokens, ~$2.40 per 1M output tokens
- This MVP will use minimal tokens in testing

---

## 2. Supabase Project (Required)

### Create Project
1. Go to https://supabase.com
2. Click "Start your project"
3. Sign in with GitHub or email
4. Click "New project"
5. Choose project name (e.g., "life-journal-mvp")
6. Set password (save this!)
7. Choose region (pick closest to you)
8. Click "Create new project" (wait ~2 min for initialization)

### Get API Keys
1. In project, go to "Settings" → "API"
2. Copy these values:
   - **Project URL** → `SUPABASE_URL`
   - **Anon Key** → `SUPABASE_ANON_KEY`
   - (Ignore "Service Role Key" for now)

### Add to .env
```bash
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Cost
- Free tier: perfect for MVP testing
- Up to 500MB storage, 1GB bandwidth/month

---

## 3. Google OAuth (Optional - Can Test with Fixtures)

### Create OAuth Credentials
1. Go to https://console.cloud.google.com
2. Create new project or select existing
3. Enable "Google+ API":
   - Search "Google+ API"
   - Click "Enable"
4. Create OAuth 2.0 credentials:
   - Go to "Credentials" (left sidebar)
   - Click "Create Credentials" → "OAuth client ID"
   - Choose "Web application"
   - Name it "Life Journal"
   - Add authorized redirect URI:
     ```
     http://localhost:3001/auth/callback/google
     ```
   - Click "Create"
5. Copy the credentials:
   - **Client ID** → `GOOGLE_OAUTH_CLIENT_ID`
   - **Client Secret** → `GOOGLE_OAUTH_SECRET`

### Add to .env
```bash
GOOGLE_OAUTH_CLIENT_ID=xxxx.apps.googleusercontent.com
GOOGLE_OAUTH_SECRET=GOCSPX-xxxxxxxxxxxx
```

### Cost
- Free forever for OAuth (no charges)

## 4. Amazon API Keys (Optional - Can Use Test Fixtures)

### Option A: Use Plaid Instead (Easier)
Instead of direct Amazon API, use Plaid to get spending data:

1. Go to https://plaid.com
2. Sign up for free account
3. Get your credentials from dashboard
4. Use same approach for other financial integrations

### Option B: Direct Amazon API
1. Go to https://developer.amazon.com
2. Create account
3. Register as seller
4. Get Product Advertising API credentials

### For MVP Testing
- Skip this for now, use test fixtures
- Ralph mode will generate mock Amazon data
- Add when you want real data

## 5. Vercel (Optional - For Backend Deployment)

### Create Account
1. Go to https://vercel.com
2. Sign in with GitHub (recommended)
3. Create new project when prompted

### Get Deployment Token
1. Go to Account Settings → "Tokens"
2. Create new token
3. Copy token → `VERCEL_TOKEN` in .env

### Cost
- Free tier: perfect for MVP
- 100GB bandwidth/month, unlimited deployments

## Complete .env File Template

After getting all keys, your `.env` should look like:

```bash
# ============================================
# Life Journal MVP - Environment Variables
# ============================================

# Supabase (REQUIRED)
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Claude API (REQUIRED)
CLAUDE_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxx

# Google OAuth (OPTIONAL - can test with fixtures)
GOOGLE_OAUTH_CLIENT_ID=xxxx.apps.googleusercontent.com
GOOGLE_OAUTH_SECRET=GOCSPX-xxxxxxxxxxxx

# Server
NODE_ENV=development
PORT=3001
BACKEND_URL=http://localhost:3001

# These are auto-generated by setup.sh - leave them
JWT_SECRET=<auto-generated>
ENCRYPTION_MASTER_KEY=<auto-generated>

# Optional - for deployment
VERCEL_TOKEN=<your-vercel-token>
```

## Troubleshooting

**"Can't find my API key"**
- Supabase: Settings → API (Project Settings, not account settings)
- Claude: console.anthropic.com → API keys
- Google: console.cloud.google.com → Credentials

**"Rate limited"**
- Free tiers have rate limits
- Contact support to increase limits
- For MVP testing, limits are very generous

**"API key not working"**
- Make sure .env file is saved
- Make sure you're in the right project/account
- Double-check no extra spaces in the key
